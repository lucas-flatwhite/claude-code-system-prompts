# Simplify 스킬

> **유형**: 번들 스킬 (Claude Code 바이너리에 컴파일됨)
>
> **호출 방식**: `/simplify` 또는 `/simplify focus on [area]`
>
> `/simplify` 명령은 최근 변경된 파일에 대해 자동화된 멀티 에이전트 코드 리뷰와 정리 작업을 수행하는 내장 스킬입니다. 기능 구현이나 버그 수정 후, pull request를 열기 전에 실행하도록 설계되었습니다.

## 사용 가능 범위

이것은 **번들 스킬**이므로 Claude Code 바이너리에 포함되어 있으며 별도 설치가 필요 없습니다. `.claude/skills/`에 두는 사용자 정의 스킬과 달리, 이 스킬은 Anthropic이 유지보수하며 Claude Code 릴리스와 함께 갱신됩니다. 정확한 `SKILL.md` 원본은 바이너리에 컴파일되어 있어 공개적으로 접근할 수 없습니다.

아래 문서는 공개 문서, 커뮤니티 자료, 동작 분석을 바탕으로 재구성한 것입니다.

## 워크플로

### 1단계: 변경 탐지

이 스킬은 최근 변경사항을 검사해 리뷰 범위를 파악합니다.

- Git 저장소 안에서는 `git diff`(또는 스테이징된 변경에는 `git diff HEAD`)를 실행해 무엇이 수정됐는지 판단합니다.
- Git 저장소가 감지되지 않으면 가장 최근에 수정된 파일을 검토합니다.
- diff 출력이 리뷰의 정확한 범위를 정의합니다.

### 2단계: 3에이전트 병렬 리뷰

세 개의 특화된 서브 에이전트가 병렬로 생성되며, 각 에이전트는 컨텍스트를 위해 전체 diff를 전달받습니다.

#### 코드 재사용 에이전트

다음 항목을 식별하기 위해 변경사항을 검토합니다.

- 공용 함수로 추출할 수 있는 중복 로직
- 새 코드를 쓰는 대신 활용해야 하는 기존 유틸리티 함수나 헬퍼(helper)
- 불필요한 코드 블록과 반복 패턴
- 재사용 가능한 컴포넌트나 모듈로 리팩터링할 기회

#### 코드 품질 에이전트

코드 구조와 스타일을 평가합니다.

- 네이밍 일관성과 가독성
- 함수 분해 수준과 제어 흐름의 명확성
- `CLAUDE.md`에 정의된 코딩 표준 준수 여부
- 누수 추상화(leaky abstraction), 문자열 중심 코드(stringly-typed code), 불필요한 중첩 같은 코드 스멜
- 과도한 설계, 불필요한 추상화, 성급한 최적화("gold-plating")

#### 효율성 에이전트

성능과 리소스 사용을 분석합니다.

- 불필요한 할당과 중복 계산
- 배치 처리 또는 최적화할 수 있는 루프
- N+1 쿼리 패턴과 비효율적인 파일/네트워크 접근
- 놓친 동시성 활용 기회
- 불필요한 리렌더링 (프론트엔드 코드)

### 3단계: 집계 및 수정 적용

세 에이전트가 모두 리뷰를 마치면 다음을 수행합니다.

1. 발견 사항을 집계하고 중복을 제거합니다.
2. 오탐(false positive)이나 대응 가치가 낮은 사항은 건너뜁니다.
3. 수정 사항을 코드베이스에 직접 적용합니다.
4. 무엇이 바뀌었는지 요약해 사용자에게 보고합니다.

## 사용법

```
/simplify                              # Review all recent changes
/simplify focus on memory efficiency   # Focus on specific concern
/simplify focus on performance         # Target performance issues
```

## CLAUDE.md와의 통합

`/simplify`의 효과는 프로젝트의 `CLAUDE.md` 파일에 크게 좌우됩니다. 품질 에이전트는 그 안에 정의된 코딩 표준과 선호를 사용해 제안을 팀 컨벤션에 맞춥니다. `CLAUDE.md`가 잘 정리된 프로젝트일수록 결과가 훨씬 좋아집니다.

## 설계 특성

| 속성 | 세부사항 |
|----------|--------|
| **Agent count** | 병렬 서브 에이전트 3개 |
| **Scope** | 최근 변경사항만 (`git diff` 기준) |
| **Execution** | 효율성을 위해 에이전트를 병렬 실행 |
| **Fix mode** | 요약과 함께 자동 적용 |
| **False positives** | 논쟁하지 않고 건너뜀 |
| **Customization** | 특정 focus 영역을 인자로 받을 수 있음 |
| **Dependency** | 프로젝트별 표준을 위해 `CLAUDE.md`를 읽음 |

## 워크플로 내 위치

권장 워크플로는 다음과 같습니다.

```
Build feature → Run tests → /simplify → Review diffs → Commit
```

즉, `/simplify`는 구현 후 정리 단계로서, 장황해지기 쉬운 AI 생성 코드를 다듬는 역할을 합니다. 인간 코드 리뷰를 대체하는 것은 아니며, 그중 기계적인 정리 부분을 자동화합니다.

## 관련 명령

| 명령 | 관계 |
|---------|-------------|
| `/init` | `/simplify`가 표준을 읽을 `CLAUDE.md`를 설정 |
| `/diff` | `/simplify` 실행 전 변경사항 미리 보기 |
| `/batch` | 전체 코드베이스에 걸친 대규모 리팩터용 (`/simplify`는 최근 변경만 대상으로 함) |
